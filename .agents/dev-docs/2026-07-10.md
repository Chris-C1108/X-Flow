## 2. 首屏渲染性能优化 (Skeletons to Cards Delay)

- **字体异步加载**：将 Google Fonts 样式链接改为异步非阻塞加载 (`media="print" onload="this.media='all'"`），首屏不再阻塞等待网络字体包下载。
- **探测避让**：将后台连通性检测（针对 twitter CDN 的双链路 HEAD 探测）延时 3 秒运行，防止其与首屏关键数据接口竞争 TCP 连接资源。
- **持久化本地缓存**：将 `CacheManager` 改为持久化缓存（保存到 `GM_setValue`），在 5 分钟 TTL 内再次访问可实现 `<5ms` 缓存直接渲染，消除首屏骨架屏加载延时。

## 3. 卡片布局与交互体验升级 (Card UX)

- **无感纯净卡片**：隐藏了视频卡片上的播放和暂停图标，提供极简海报式布局。
- **缓冲防黑屏过渡**：为自动静音播放（No.1卡）、悬停预览（PC）与长按预览（Mobile）视频元素监听 `playing` / `timeupdate` 事件。在视频未开始解码渲染前，保持封面图 100% 不透明度；首帧渲染完毕后才渐隐封面，杜绝黑屏/暗屏过渡。
- **移动端单触直达播放器**：优化 `touchend` 事件，长按触发预览后抬起手指，立刻以当前预览的 `currentTime` 时间戳无缝打开 full-screen 抖音模式播放器，彻底解决由于移动端 `preventDefault` 导致需要二次点击才能进入播放的 bug。

## 4. 播放器视频缓冲限速优化 (Buffering Performance)

- **注销 Host Service Worker**：
  - **核心痛点**：宿主站点后台的 Service Worker 会恶意拦截 `video.twimg.com` 媒体流的分片 Range 请求，导致多次握手重试，缓冲延迟飙升至 10-15s。
  - **方案**：在 Sandbox 初始化阶段强行注销宿主域名的所有 Service Worker，彻底绕过拦截层，恢复 5-6s 的原生极速下载。
- **并发调度让路**：
  - 延迟加载邻近节点（当前 index - 1 / + 1）与前置 Prefetch API 详情请求 1.5 秒。在打开播放器和滑动时，全力供给当前活动视频的流媒体分片连接。
- **修复预加载 Bug**：
  - 修复了 `preloadNode` 只改变 `video.preload = 'auto'` 而未绑定 `video.src` 的缺陷，确保后台数据流真正开始在浏览器中缓冲缓存。
