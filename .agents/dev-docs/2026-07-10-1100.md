# X-Flow v6.2.1 开发日志 — 2026-07-10

## 版本概要
* **版本号**: `6.2.1`
* **日期**: 2026-07-10 11:00 (本地时间)
* **核心更新**: 修复流媒体播放器在快速滑动时的复用漏洞与进度条残留问题，实现断点续播与无延迟预加载机制，解决移动端 WebKit 第一点击 hover 阻断 Bug。

---

## 优化与 Bug 修复详情

### 1. 快速滑屏“永远循环那几个视频”Bug 修复 (TikTokMode.ts)
* **问题**: 列表切换的 `loadNode` 逻辑被包裹在 1.5 秒延迟的 `preloadTimer` 中。如果用户滑动速度快于 1.5 秒，每次切换都会清空该定时器，导致物理节点的 `data-index` 和 `src` 从未获得更新机会，用户在 3 个物理节点中不断循环初始缓存的视频。
* **修复**: 在 `navigate` 切换索引阶段，**立即同步调用** `this.loadNode(this.currentIndex)`。卡片封面图和媒体流会在第一瞬间完成对应索引的初始化绑定。

### 2. 视频切换时进度条残留/重置延迟 Bug 修复 (TikTokMode.ts)
* **问题**: 切换视频后虽然手动调用了重置进度条为 0，但前一个视频在 `pause()` 或卸载时会由浏览器异步抛出最后一两个 `timeupdate` 回调。该事件处理程序将上一个视频最后的播放进度强行覆盖到了已经被清零的进度条上，导致加载新视频时视觉卡顿。
* **修复**: 为 `<video>` 的关键事件监听器（`ontimeupdate`, `onended`, `onleavepictureinpicture`）加上了**索引一致性守护**（Index Guard）：
  `if (video.getAttribute('data-index') !== this.currentIndex.toString()) return;`
  这能确保所有属于已滑过视频的晚到异步事件都会被直接忽略，进度条处于完美的 0 填充加载状态。

### 3. 播放进度缓存与恢复功能实现 (ProgressManager.ts & TikTokMode.ts)
* **设计**: 新增 `ProgressManager` 播放进度管理器。通过 `Map` 在内存中缓冲实时进度（高频 `timeupdate` 更新），以 2 秒为间隔节流持久化写入 `localStorage`，并在视频暂停、切走和关闭播放器时执行强制同步。
* **清理机制 (LRU + TTL)**:
  * 限制进度最大存储为 `200` 条记录，超出时自动淘汰最久未看的记录。
  * 生存期（TTL）为 `7 天`，每次读取或写入时过滤超期记录。
  * 播放进度在剩余小于 `2 秒` 内时自动清除（视为已看完），下次进入从头播放。
* **瞬间还原**: 在进入新视频时，若存在缓存进度，会在 `loadedmetadata` 时期通过 `video.currentTime` 瞬间seek。同时，如果缓冲开始前已知时长，会在界面上瞬间将进度条和时间文本提前渲染至缓存百分比，防黑屏期间的视觉闪烁。

### 4. 彻底解决回滑重新缓冲延迟 (VirtualList.ts & TikTokMode.ts)
* **问题**: `3-Node VirtualList` 只能保存当前及前后邻近共 3 个节点的媒体缓冲。若向下滑动 2 步，最顶上的视频就会被回收挪用以加载新视频。当用户重新回滑 2 步时，视频已被完全卸载，浏览器不得不发起 `Range` 请求重新下载切片，造成 3 秒以上的黑屏延迟。
* **修复**: 将物理 DOM 节点池的容量由 **3 提升至 5**。允许同时在 DOM 树中保留当前视频前后各 2 个节点的媒体资源。当用户回滑 1-2 步时，之前的视频和缓冲全部存活，开播延迟降为 0。
* **内存防御**: 为了防止 5 节点池在播放器关闭时持续占用内存和硬件解码器，在 `closeModal` 中调用 `unloadAllVideos()`，对全部 5 个视频执行 `removeAttribute('src')` 和 `load()`，确保退出播放器时 0 内存占用。

### 5. 流媒体加载与建连性能优化 (TikTokMode.ts)
* **预解析提速**: 移除了 1500ms 定时器对地址预解析的限制，`startPrefetching` 变为**零延迟启动**。后台会在低带宽下优先预解析后续 5 个视频的 HTML 和 302 重定向 CDN URL。当用户划入新视频时，播放地址早已准备妥当。
* **连接预热**: 邻近节点在后台预解析出真实 URL 后，立即赋予 `<video>` 标签，初始设为 `preload="metadata"`。使得浏览器在后台提前完成推特 CDN 的 DNS 查询、TCP 握手及 SSL 密钥协商。

### 6. 移动端“首击卡片仅缩放、不进播放器”Bug 修复 (Layout.ts)
* **问题**: 移动端 WebKit (如 iOS Safari) 检测到卡片配置了 `.media-card:hover` 及父容器的 `mouseenter` 捕获后，首击卡片会判定为“鼠标悬停 (Hover)”仅展示缩放，不派发 `click` 事件。
* **修复**: 在 `touchend` 监听中劫持短按手势（未发生滚动且无长按预览时），立即调用 `openModal` 并在内部执行 `e.preventDefault()` 彻底截断后续模拟鼠标/Click 事件。不仅彻底解决了 Safari Hover 的首击失效问题，还顺带清除了移动端原生的 300ms 点击延迟，点击反馈极为清脆跟手。
