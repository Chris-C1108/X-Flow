## Hotfix Log

- ✅ **2026-04-18 — appRoot 缺失自愈热修复**
  - `Sandbox.initialize()` 不再假定 `#xflow-app-root` 一定存在，缺失时立即重建 `body + appRoot` 后继续启动。
  - 修复首轮启动在 `.dataset` 处因 null 崩溃的问题，避免 finally 移除 splash 后留下无 UI 页面。

- ✅ **2026-04-18 — document-start 晚接管热修复**
  - 在构建输出顶部注入超早期 preboot banner，先于主 bundle 盖住宿主站首屏并显示 `X-FLOW LOADING`。
  - 记录静态 import 会拖慢 `document-start` 接管的经验，但保留单文件 userscript 构建，不引入额外 chunk 依赖。

- ✅ **2026-04-18 — 启动黑屏 / Splash 卡死热修复**
  - `Sandbox.initialize()` 通过 `finally` 强制关闭 splash，并在 `appRoot` 上写入 `booting / ready / failed` 启动状态。
  - 启动入口统一改为显式 `initialize().catch()`，首轮异常会立刻移除 splash，避免 Promise rejection 被全局抑制后留下纯黑遮罩。
  - 恢复逻辑不再把“splash 还在”或“appRoot 有子节点”视为成功，而是按 splash 超时窗口和 `data-xflow-state` + app shell 完整性联合判断。

- ✅ **2026-04-18 — 宿主 DOM 篡改恢复热修复**
  - 将启动入口改为显式 `initialize().catch()`，避免首轮启动异常被全局 `unhandledrejection` 抑制逻辑吞掉。
  - 不再仅依赖 `appRoot.children.length`，改为检查 `.app-layout` + `#main-scroll` 是否完整存在。
  - 新增 `MutationObserver` + `load/pageshow` 兜底，宿主晚到脚本清空 DOM 时立即重建 app shell 并重新注入。

- ✅ **2026-04-17 — Hero / Splash 稳定性热修复**
  - Hero Carousel 只允许当前可见卡片持有 `src` 并播放，隐藏卡片统一卸载，避免和 TikTok 播放器争抢带宽。
  - 打开播放器前强制卸载全部 Hero 预览视频，修复进度跳转时“加载慢 / 有声音但画面不动”的高概率诱因。
  - 在 `document.open()` 前注入 preboot veil，进一步压低源站页面偶发闪现。

## M1 — 播放器功能补全 Phase 1（快速提升）

> **目标：** 补全最高频需求且开发成本最低的功能

### 任务清单

- ✅ **M1-1** 倍速播放菜单
  - **位置：** `TikTokMode.ts` 顶栏区域新增速度按钮
  - **交互设计：** 点击弹出 `[0.5×, 1×, 1.5×, 2×]` 浮层，选中后显示当前倍速标签
  - **实现核心：** `video.playbackRate = selectedRate`
  - **持久化：** 通过 `Storage.ts` 跨 session 记住用户偏好
  - **CSS：** 复用现有 `tm-settings` 面板样式，新增倍速选项行

- ✅ **M1-2** 画中画（Picture-in-Picture）
  - **位置：** 顶栏区域新增 PiP 按钮（隐藏 if `!document.pictureInPictureEnabled`）
  - **交互设计：** 点击进入/退出 PiP，PiP 窗口关闭时自动同步回 modal 播放状态
  - **兼容性：** iOS Safari 不支持，需检测 `document.pictureInPictureEnabled` 条件渲染
  - **事件监听：** `video.onleavepictureinpicture` 处理 PiP 关闭回调

- ✅ **M1-3** 暂停中心提示动画
  - **需求来源：** `todo/002.md` — "暂停播放需要有短暂提示"
  - **交互设计：** 点击暂停/恢复时，中心短暂显示 ▶/⏸ 图标（已有 `.tm-center-icon` CSS，仅需接线）
  - **实现：** 在 `togglePlayCurrent()` 中触发 `classList.add('show')` + `setTimeout remove`

- ✅ **M1-4** 跳转视频作者主页
  - **位置：** actions sidebar 新增"主页"按钮（作者头像 + "主页"文字）
  - **数据来源：** API 返回数据中的 `author_url` 或 `author_id` 字段
  - **交互设计：** 点击在新标签页打开（`window.open` or `GM_openInTab`）
  - **降级：** 若 API 无 `author_url` 字段则隐藏此按钮

**M1 验收标准：** 倍速切换流畅无卡顿；PiP 进出无黑屏；暂停提示动画自然；跳转作者页正常。

> **目标：** 进阶交互手势 + 与数据链路的视觉呈现

### 任务清单

  - **视觉反馈：** 显示 `◀ 10s` / `10s ▶` 动画提示

- ✅ **M2-3** 进度条高光时刻打点

- ✅ **M2-4** 评论入口（占位设计）
  - **需求来源：** `todo/002.md` — "评论"
  - **本期实现：** actions sidebar 新增评论图标按钮（点击暂时 `alert("评论功能开发中")`）
  - **预留扩展：** 设计好组件槽位，v6.1 填充真实评论 sheet
  - **注意：** 由于外部站点评论系统不可控，此功能可能需要接入外部服务（v6.1 评估）

**M2 验收标准：** 长按加速无误触；双击定位准确显示反馈；高光点渲染无性能抖动。

## 🔧 技术约束 & 注意事项

### 不可违背的边界（来自 AGENTS.md Experience Index）

| 约束 | 相关 KI |
|------|---------|
| 绝对不在 `document.open()` 之后用 `@require` 加载外部库 | KI #11/#19 |
| 视频 referrer 控制必须通过 `<meta>` 文档级，不能用元素级 `referrerpolicy` | KI #04 |
| CSS base styles 必须在 `@media` 之前声明 | KI #07 |
| 含模板字符串的文件修改后必须 `npm run build` 验证 | KI #02 |
| Channel 切换通过 `isAnimeOnly` 参数，绝不改 `baseUrl` | KI #06 |

### 新增技术决策

| 决策 | 理由 |
|------|------|
| 不引入任何第三方播放器库（ArtPlayer/XGPlayer/Swiper）| CSP + document.open() + 混淆 三重系统级阻断 |
| 推荐抽屉优先异步加载，不阻塞播放主流程 | 用户感知的加载速度 > 推荐数据实时性 |
| 高光 marker 使用 CSS 定位，不用 Canvas | Canvas 在混淆后有额外 API 兼容性风险 |
